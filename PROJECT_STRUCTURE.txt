## PROJECT FILE MANIFEST

This file provides a compact, LLM-friendly overview of the project's file structure.

### Documentation & Meta

- `AI_CONTEXT.md`: System protocol and context for the AI assistant, defining architectural rules, workflow, and current project state.
- `HISTORY.md`: Chronological development history, tracking features completed in each phase.
- `microtonal_implementation_guide.md`: Technical design document for implementing microtonal harmony.
- `PROGRESS_VISION.md`: High-level product vision, feature roadmap, and music theory concepts.
- `README.md`: Project overview, feature list, development roadmap, and local setup instructions.
- `PROJECT_STRUCTURE.txt`: This file. A manifest of all project files and their purpose.

### Core Application & State

- `progressmain.js`: Main application controller. Initializes all modules and sets up top-level event listeners.
- `init.js`: Entry point script loaded by `index.html`. Handles polyfills and loads the main `progressmain.js` module.
- `config.js`: Central configuration file containing all static constants and magic numbers.
- `server.js`: A lightweight Node.js server for local development.
- `store.js`: Manages the global application state (SSOT), history (undo), and section/song data structures.
- `storage.js`: Handles `localStorage` `saveState` and `loadState` operations.
- `stateUtils.js`: Pure utility functions for complex state transformations (e.g., calculating swaps on reorder).
- `db.js`: Manages IndexedDB for storing and retrieving custom user drum samples.

### Music Theory & Generation

- `theory.js`: Core music theory engine. Pure functions for chord generation, voice leading, tension analysis, etc.
- `chordDictionary.js`: Data module containing the dictionary of chord symbols and their corresponding semitone intervals.
- `bassGenerator.js`: Generates intelligent bassline patterns that algorithmically respond to drum and chord rhythms.
- `arp.js`: Generates arpeggiated note sequences based on style, rate, and gate settings.

### UI & Rendering

- `ui.js`: Renders the main progression display, synesthetic colors, and the chord inspector panel.
- `inspectorController.js`: Renders and manages the "Inspector" panel for a selected chord.
- `modalController.js`: Manages the AI Prompt and Settings modals.
- `songController.js`: Manages the "Song Mode" UI (section tabs, macro-sequencer tray) and its interactions.
- `transportController.js`: Manages the floating action button (FAB) for play/pause and volume control.
- `dragdrop.js`: Handles all drag-and-drop logic for the UI, including reordering items, adding new items, and positioning loop brackets.
- `clipboardUtils.js`: Handles copy/paste functionality for rhythm patterns.

### Rhythm Editor

- `rhythmEditor.js`: Main controller for the Pattern Editor panel. Orchestrates state and rendering for its sub-modules.
- `rhythmRenderer.js`: Top-level renderer for the timeline, delegating to `drumRenderer` or rendering slice instances.
- `rhythmControls.js`: Initializes all toolbar controls within the rhythm editor (copy, paste, grid, etc.).
- `arpControls.js`: Initializes UI controls for applying arpeggiator settings.
- `bassControls.js`: Initializes UI controls for the bassline generator.
- `timelineEditor.js`: Handles user interactions (drag, resize, slice) for the generic slice-based timeline.
- `drumEditor.js`: Handles user interactions (add, move, delete hits) for the drum grid.
- `drumRenderer.js`: Renders the drum grid UI, including grid lines, hits, and custom sample waveforms.
- `patternUtils.js`: Utility functions for creating and manipulating rhythm patterns (e.g., `sliceInstance`, `addDrumHit`).
- `patternDrawMath.js`: Pure mathematical logic for the "Pencil Draw" tool's boolean carving operations.
- `patternResolver.js`: Logic to resolve global vs. local patterns, handling inheritance modes like 'loop' or 'stretch'.
- `rhythmConfig.js`: Configuration constants for the rhythm editor (e.g., `GRID_STEPS`, `DRUM_ROWS`).

### Audio & Export

- `synth.js`: Manages the Web Audio API engine, including the master compressor, gain nodes, and scheduling tones.
- `sequencer.js`: Handles the main playback loop, scheduling notes and UI highlights with `requestAnimationFrame`.
- `synthEngines.js`: A registry of synth voices (e.g., 'sawtooth', 'sine', 'fm') for the Web Audio API.
- `drumEngines.js`: A registry of synth voices for synthesized drum sounds (kick, snare, etc.).
- `exportController.js`: Manages the UI for MIDI, WAV, and Stems export, including the draggable switch.
- `exportStateBuilder.js`: Compiles and "bakes" the application state (including macro-sequences) into a flattened format suitable for export.
- `midi.js`: Handles the generation and export of `.mid` files using `MidiWriterJS`.
- `wavExport.js`: Handles offline rendering and export of `.wav` audio files using `OfflineAudioContext`.
- `wavEncoder.js`: Utility to convert a raw `AudioBuffer` into a valid `.wav` file binary.
- `promptGenerator.js`: Generates text prompts detailing harmonic structure for AI music generators.

### Styling & Assets

- `index.html`: The single HTML entry point for the application. Loads all CSS and JS modules.
- `style.css`: Main stylesheet for global layout, controls, and component styling.
- `theme.css`: Contains CSS variables for light and dark themes.
- `drum-grid.css`: Styles specific to the drum grid editor.
- `rhythm-editor.css`: Styles for the rhythm editor panel, timeline, and instances.
- `song-mode.css`: Styles for the song tray and section tabs.
